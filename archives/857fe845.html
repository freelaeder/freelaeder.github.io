node.js是什么？

  • node.js不是一门语言，不是库，不是框架，node.js是一个运行javaScript运行时的环境。
  • node.js没有Bom，Dom；
  • 作用：文件读写，网络服务的构建，网络通信，HTTP服务器等

  • 特性：event.driven事件驱动，ligtweight and efficient 轻量和高效，non-blocking I/o非阻塞io模型

    浏览器中的JavaScript组成

  • Chrome 浏览器 => V8
  • Firefox 浏览器=> OdinMonkey(奥丁猴)
  • Safri 浏览器=> JSCore
  • IE 浏览器=> Chakra(查克拉)

  • etc… 其中，Chrome 浏览器的 V8 解析引擎性能最好!

    注意

  • 运行环境，指 代码能够正常执行，所必须的条件浏览器，就是 JS 代码的 运行环境。（包括解析引擎、窗口显示结果……）脱离运行环境，JS 代码将会失去作用。
  • 浏览器运行环境中，能够执行的 JS 代码包括（ECMAScript、WebAPI）ECMAScript 肯定能够运行，因为它是核心语法
    WebAPI 是浏览器环境特有的，只能在浏览器环境中才能使用。
  • 交互模式(了解)
  • REPL(Read-Eval -Print-Loop)交互模式指的是在终端窗口中，执行简单的JavaScript代码
  • 操作步骤：打开任意终端，直接输入 node 命令并回车执行你的JS代码，按回车表示执行按两次“Ctrl+C”退出
  • 只适合少量js代码
    [tag type=”success” outline=”1”]脚本模式(重要哦)[/tag]
  • 如果有大段的JS代码需要在Node环境中运行，可以把JS代码写到JS文件中终端中，使用 node xxx.js 命令即可运行文件中的代码
  • 注意哟：两种模式不能混用，如果使用第一种模式，必须先退出，然后才能用第二种
    [tag type=”primary” outline]node中的模块化[/tag]
  • Node中的模块化，就是把一个大的JS文件拆分成若干个小文件，还能按照一定的语法组装到一起。
  • 规定，用户创建的每个JS文件，就是一个小模块，叫做自定义模块
  • 我们可以按照一定的语法将这些小模块组合到一起，形成一个完整的项目
    [tag type=”success” outline=”1”]Node中的模块分类[/tag]
  • 内置模块（核心模块）：Node安装后，即可使用的模块，Node环境自带。
  • 自定义模块：用户自己创建的每个JS文件，都是自定义模块
  • 第三方模块：其他人或公司、组织开发的模块，发布到 npm 网站，我们需要下载使用的模块
    [tag type=”primary” outline]加载模块的语法[/tag]
    1.1加载自定义模块
  • a 文件，必须通过 module.exports 共享（导出、暴露）当前模块中的变量
  • b 文件，需要通过 require() 加载（导入）。（a 文件导出什么，b 文件就得到什么）
  • 加载自定义模块时，必须要带路径 （相对路径、绝对路径都可以；但必须带路径，哪怕是 ./ 也不能省略）
  • 加载自定义模块时，可以省略后缀
    1.2加载内置模块
  • 直接使用 require() 加载即可。
  • 加载模块时，无需带路径，比如 require(‘fs’)
  • 加载内置模块，得到对象类型，对象中内置了很多API方法
    1.3加载第三方模块
  • // npm （node package manager）,npm包管理器
  • 第三方模块，发布在https：//www.npm.js.com
  • 1,npm init 初始化package.json || npm init -y
  • 2,安装npm install 包名 || npm i 模块名
  • 3,卸载npm uninstall 包名 || npm un 模块名
  • 可以在文件package.json中的dependencied查看安装的模块
