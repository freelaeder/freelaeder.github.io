[hint]Http协议是什么？[/hint]

  • http协议（ hypertext transport protocol协议）【超文本传输协议】，详细规定了浏览器和万维网之间通信的的规则 ；
    [tag type=”primary” outline]常见状态码&含义[/tag]
  • 200 - 请求成功，已经正常处理完毕
  • 301 - 请求永久重定向，转移到其它URL
  • 302 - 请求临时重定向
  • 304 - 请求被重定向到客户端本地缓存
  • 400 - 客户端请求存在语法错误
  • 401 - 客户端请求没有经过授权
  • 403 - 客户端的请求被服务器拒绝，一般为客户端没有访问权限
  • 404 - 客户端请求的URL在服务端不存在
  • 500 - 服务端永久错误
  • 100-199 用于指定客户端应相应的某些动作。
  • 200-299 用于表示请求成功。
  • 300-399 用于已经移动的文件并且常被包含在定位头信息中指定新的地址信息。
  • 400-499 用于指出客户端的错误。
  • 500-599 用于支持服务器错误。

[tag type=”danger”]Ajax 概述[/tag]
​Ajax：标准读音 [ˈeɪˌdʒæks] ，中文音译：阿贾克斯
它是浏览器提供的一套方法，可以实现页面无刷新更新数据，提高用户浏览网站应用的体验。
[tag type=”success”]Ajax 应用场景[/tag]

  • 页面上拉加载更多数据
  • 列表数据无刷新分页
  • 表单项离开焦点数据验证
  • 搜索框提示文字下拉列表

[tag type=”warn” outline]JSON[/tag]

  • JSON是什么
  • JSON ( JavaScript Object Notation) ，是一种数据交互格式。
  • 在真实的项目中，服务器端大多数情况下会以 JSON对象作为响应数据的格式。当客户端拿到响应数据时，要将 JSON 数据和 HTML 字符串进行拼接，然后将拼接的结果展示在页面中。

  • 在 http 请求与响应的过程中，无论是请求参数还是响应内容，如果是对象类型，最终都会被转换为对象字符串进行传输。

  • 注意事项

  • JSON 规定字符集是UTF-8，字符串必须使用””，Object 的键也必须使用””
  • 数组或对象的最后一个成员，不能加 逗号

[tag type=”info”]git常用命令[/tag]

  • 初始化
  • git init 初始化配置，此时会出现一个隐藏的文件（.git）
  • git config —global user.name “xxx” 配置用户名
  • git config —global user.email “xxx@XXX.com” 配置邮箱
    [tag type=”info” outline]查看状态[/tag]
  • git status 查看当前版本状态（是否修改）
  • git add . 增加当前的更改过的文件 提交到暂存区
  • git commit -m “提交仓库” 并加以注释
  • git commit —amend -m “XXX” 合并提交，
  • pwd 查看此时在那个文件夹中
    [tag type=”primary” outline]查看记录[/tag]
  • git log -n 查看最近n次的提价
  • git log —oneloine 单行查看简陋版日志
  • git —reflog —oneline 简陋日志
  • git relong 查看当前版本之后的日志，包括版本回退的记录
  • git log 查看当前的日志，不包括版本回退的记录
  • git reset —hard 版本号 回退当前的版本号，
  • 新增文件 each 测试>demo.txt 此时文件会多出此文件，内容为测试
  • cat demo.txt 查看demo.txt的内容
  • vi demo.txt 进入vi编辑器
    [tag type=”info” outline]上传本地文件至GitHub[/tag]
  • 1 首先右键你的项目，如果安装git成功，右键会出现两个新的选项，Git Gui Here ，Git Bash Here ，选择后者
  • 2 把github上面的仓库克隆到本地， git clone 仓库地址
  • 3 此时会多出一个文件夹，把需要上传的文件复制到此文件夹
  • 4 输入 cd 此文件夹的名字 ，进入此文件夹
  • 5 输入命令 git add . 注意点号，不能省略，此操作是把修改的内容添加进来
  • 6 输入 git commit -m “提交信息” （项目的说明）
  • 7 若出现 **Please tell me who you are ,说明没有配置用户，配置用户命令即可，再次执行第6步
  • 8 最后 git push -u origin master (把本地仓库上传至github，需要输入密码)；
